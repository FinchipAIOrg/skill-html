Decision notes

Core value

Turns execution into a controlled, auditable workflow rather than a simple order-entry prompt. The best use case is large crypto order execution where slippage, maker participation, and account safety matter.

Primary risk

This is a high-permission trading skill. Production use requires operator confirmation discipline, exchange API permission hygiene, IP whitelist maintenance, and clear post-trade monitoring.

Must-test samples

Use a read-only account inventory task, a simulated TWAP plan, a PERP margin check, a paired order rollback scenario, a TCA export, and an error recovery case before listing.

Failure signal

Mark as failed if it invents balances, skips scripts, hides stderr, submits without confirmation, loses pagination, or gives investment recommendations instead of execution operations.

Evaluation standard integration

How to evaluate it under Gate, Score, and Operate

Gate The skill must demonstrate incremental capability beyond a generic trading prompt: QE account discovery, exchange-specific balance routing, algorithm parameters, confirmation gate, and TCA outputs tied to scripts. Fails if it only gives generic TWAP advice, uses unsupported venues, or cannot route to the right script.
Weighted score Score weights execution coverage, write-operation safety, parameter validation, account and margin checks, TCA formula correctness, output integrity, and operational stability. A successful-looking answer with missing pre-checks or missing confirmation should score low.
Three-run stability Run the same read-only and planning tasks three times. The chosen scripts, parameter requirements, safety gate, and missing-data language should remain stable. Different runs should not randomly switch between TWAP, POV, paired-order, or margin routes without user intent.
Agent output Outputs must be reusable by another operator: final parameters, command path, confirmation state, returned IDs, pagination, raw error, next check, and exact report path. Prose-only output without order IDs, command context, or TCA fields is not enough.
Cost and safety Benchmark must measure not only success but also the cost of current data calls, report generation, credential hygiene, and the safety of local Excel outputs that may contain trading history. Hidden exports, QE_API_KEY/QE_API_SECRET leakage, or unbounded repeated polling are blockers.
Regression set Keep account discovery, market quote, Binance PERP TWAP plan, paired-order rollback, TCA Excel export, and raw-error recovery as recurring tests. Any future version should pass these before being shown as execution-ready.
