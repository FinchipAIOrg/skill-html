Decision Notes

Core value

Turns knowledge assets into reusable skills instead of one-off answers.

Usage reminder

Confirm source rights, estimated cost, input scope, and destination before conversion.

Must-test samples

One technical PDF, one Markdown docs folder, one EPUB, and one mixed-source batch.

Failure signal

The output is only a summary and lacks chapter index, glossary, patterns, or decision cheatsheet.

Evaluation Standard Integration

Gate, score, and operate checks

Standard areaThis benchmark now checksReject or watch signals
GateThe skill must confirm source format, user rights, extraction quality, and whether the content can become an installable skill rather than only a summary.Copyright-sensitive copying, unsupported files, or no cost/size confirmation fails the gate.
Weighted scoreScore is weighted across chapter indexing, pattern extraction, glossary quality, on-demand loading, support-file structure, generated SKILL.md usability, cost per conversion, and safety limits.A fluent synopsis with no reusable skill structure should fail even if it reads well.
Three-run stabilityConvert the same source three times and compare whether the core taxonomy, routing rules, and install instructions remain stable.Different runs producing incompatible skill layouts indicates weak extraction discipline.
Agent outputDownstream agents should receive an installable folder plan, references map, chapter-loading policy, test prompts, and known limitations.If a user still has to manually rediscover the book structure, the conversion is not operational.
Regression setUse a technical PDF, an EPUB/manual, and a mixed document folder as fixed regression inputs.Any run that copies long protected passages or skips license/provenance checks is a release blocker.
