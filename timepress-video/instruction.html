Instruction - Browser Motion Explainer

I am Timepress Video. I turn one Chinese idea into a motion story people can grasp in seconds.

This skill turns Chinese articles, viewpoints, scripts, product workflows, and visual briefs into short browser-playable motion explainers. Instead of generating unstable image-to-video frames, it keeps Chinese labels crisp in SVG, stages one visible conceptual action, and builds a deterministic HTML/GSAP project that can be reviewed, revised, and validated in a browser.

Capability map

What the skill actually does

Source digestion Extracts one to five cognitive anchors: judgment, workflow, before/after shift, bottleneck, status change, or metaphor. Turns a vague source into a motion-ready idea instead of animating every sentence.

Workflow

The safe path from source text to playable motion

1. Digest Read the source and extract one clear anchor for a short demo or three to five anchors for a multi-scene explainer.
2. Choose preset Use editorial-sketch for ideas, clean-product for workflows, data-flow for systems, and xiaohei-sketch only when requested.
3. Plan shots Write the motion shot list internally before coding. Each shot should have one conceptual action and a visible main mover.
4. Lock style Create or update DESIGN.md with palette, typography, object rules, label limits, and anti-patterns.
5. Build HTML Draw the final hero frame first with SVG, then add GSAP beats such as draw, feed, crank, press, pull, sort, output, write, and hold.
6. Verify Run the validator, open the preview, check early/middle/final frames, and deliver a directly openable link without asking the user to run commands.

Usage boundaries

What Timepress will and will not do

It delivers a browser-native master

The default output is validated playable HTML. Current examples load the GSAP runtime from a CDN; offline delivery should bundle that runtime locally. Video export remains a downstream task.

It will keep text sparse

Labels should be short, spatially attached to objects, and readable. The frame should not become a course slide.

It will not force Xiaohei

The Xiaohei preset is optional and attribution-bound. Most tasks should use editorial, product, or data-flow presets.

It will not rely on unstable animation

No Math.random(), Date.now(), infinite repeats, async timeline construction, or render-time data fetches.
