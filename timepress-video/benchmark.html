Decision notes

Core value

Timepress creates a repeatable browser-motion workflow for Chinese explainers: digest, preset, shot list, design lock, SVG/GSAP build, validation, preview.

Verified evidence

Six of six packaged examples pass the static validator and expose the required SVG, composition metadata, and paused timeline contract.

Current evidence gap

No bundled clean-product reference output proves that preset yet. Published-preview retention, offline GSAP bundling, and three-run stability also remain unverified.

Failure signal

Fail if index.html lacks metadata, SVG, paused timeline, preview mode, or if labels overlap, text is unreadable, or the final frame does not hold.

Evaluation standard integration

How to evaluate it under Gate, Score, and Operate

Gate The skill must produce more than generic animation advice: it should create DESIGN.md, index.html, inline SVG, composition metadata, a registered paused timeline, validation output, and a preview link. Fails if it only returns a storyboard, a PPT-like static diagram, or an image-to-video prompt.
Weighted score Current weights: source compression 15%, preset discipline 15%, HTML/GSAP contract 25%, validation coverage 15%, preview reliability 15%, and boundary clarity 15%. The displayed 84 is the rounded weighted result. Heavily penalize unreadable Chinese text, missing DESIGN.md, unsupported Xiaohei use, or nondeterministic animation.
Three-run stability Repeat the same short viewpoint three times. The chosen preset, output structure, metadata, validation behavior, and final preview contract should remain consistent. Random style changes, different deliverable shape, or inconsistent timeline registration are stability failures.
Agent output The final response should include the preview URL, generated file paths, validator result, and a concise note about the chosen visual preset. Vague "done" messages without preview and validation evidence are insufficient.
Cost and safety Playback is lightweight because the browser renders SVG motion, but generation cost is not measured here. Watch for unnecessary encoders, heavy assets, hidden data fetches, and runtime dependencies. Do not claim offline playback while GSAP is loaded from a CDN, and do not require the customer to run commands just to view the result.
Regression set Keep recurring tests for a conceptual viewpoint, clean-product workflow, data-flow system, Xiaohei refusal unless requested, validator pass, direct playback, and offline-runtime behavior. Every version should pass this set before being presented as production-ready.
