What you get

A browser loop an agent can reason about.

Structured observation

Snapshots return URL, title, viewport, and interactive elements with stable-in-snapshot refs such as @e2 and @ev3.

Real interaction

Open, click, fill, type, press keys, and close a persistent Chromium session from separate agent tool calls.

Visual fallback

Hybrid mode combines DOM/AX extraction with screenshot-based detection, then fuses overlapping targets using IoU matching.

Working pipeline

Observe → act → observe.

Open

Launch an isolated Chromium session and navigate to the target URL.

Snapshot

Choose AX, hybrid, or vision mode for the current page.

Resolve

Read roles, names, boxes, and refs from the structured result.

Act

Click or enter text through the ref; use keyboard actions when useful.

Verify

Take a new snapshot after mutation, then close when complete.

Ref rule: refs belong to the latest snapshot. Re-snapshot after page mutations instead of assuming an old ref still identifies the same target.

Direct CLI
vab open https://news.ycombinator.com \
  --id hn --headless

vab snapshot --id hn --mode ax
# act only if the task needs it
vab click @e2 --id hn
vab snapshot --id hn --mode ax

vab close --id hn
Mode guide

Spend vision only where it adds coverage.

AX

Fast, local, and no Anthropic key required. Best for semantic HTML, forms, and content sites.

start here

Hybrid

Runs AX and vision, then fuses results. Best for rich SPAs and mixed rendering.

complete

Vision

Screenshot-only detection for pages whose DOM semantics are not useful.

specialist
Requirements

Small surface, explicit dependencies.

  • Node.js 20.6 or newer.
  • Playwright with Chromium installed.
  • ANTHROPIC_API_KEY for hybrid or vision mode.
  • AX mode works without a model API key.

Use it for

Interactive web tasks, form workflows, navigation, UI inspection, and pages with canvas or custom-rendered controls.

Do not use it for

Static content fetches, raw API retrieval, or native desktop apps outside Chromium.
